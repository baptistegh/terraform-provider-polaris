# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Terraform provider for [Apache Polaris](https://polaris.apache.org/), the open-source implementation of the Iceberg REST Catalog specification. Allows managing Polaris catalogs, namespaces, principals, roles, and grants via Terraform.

## Commands

```bash
# Build
go build ./...

# Run all unit tests
go test ./...

# Run a single test
go test -run TestResourceCatalog ./internal/provider/

# Run acceptance tests (requires a running Polaris instance)
TF_ACC=1 POLARIS_HOST=http://localhost:8181 go test ./internal/provider/ -v -timeout 120m

# Lint
golangci-lint run

# Re-generate the management API client (after updating spec/)
go generate ./pkg/polarismanagement/

# Generate Terraform provider docs
go generate ./...

# Format
gofmt -s -w .

# Install provider locally for manual testing
go install .
```

## Architecture

This is a [Terraform Plugin Framework](https://developer.hashicorp.com/terraform/plugin/framework) provider (not the legacy SDK). All resources and data sources use the `framework` package patterns (`resource.Resource`, `datasource.DataSource`).

### Directory layout (expected)

```
main.go                        # calls providerserver.Serve
spec/
  polaris-management-service.yml  # upstream OpenAPI spec (source of truth — do not edit)
pkg/
  polarismanagement/           # generated Go client for /api/management/v1 (importable externally)
    generate.go                # //go:generate directive
    oapi-codegen.yaml          # oapi-codegen config
    client.gen.go              # generated — do not edit by hand
internal/
  provider/
    provider.go                # provider.Provider implementation, registers all resources/datasources
    providerdata.go            # ProviderData struct shared by all resources/datasources
    resource_catalog.go        # polaris_catalog resource
    resource_principal.go      # polaris_principal resource
    resource_principal_role.go # polaris_principal_role resource
    resource_catalog_role.go   # polaris_catalog_role resource
    resource_principal_role_assignment.go
    resource_catalog_role_assignment.go
    resource_catalog_grant.go
    resource_namespace_grant.go
    resource_table_grant.go
    resource_view_grant.go
    resource_policy_grant.go
    datasource_realm.go        # polaris_realm data source (no API call, reads provider config)

    datasource_principal.go
    datasource_principal_role.go
    datasource_catalog_role.go
    ...
examples/
  provider/
```

### Generated client (`pkg/polarismanagement`)

Generated from `spec/polaris-management-service.yml` using [`oapi-codegen`](https://github.com/oapi-codegen/oapi-codegen) v2. The spec covers the full `/api/management/v1` surface: catalogs, principals, principal roles, catalog roles, and grants.

The generated `Client` struct uses `RequestEditorFn` hooks for per-request customization (e.g. injecting the `Authorization: Bearer <token>` header). To instantiate:

```go
client, err := polarismanagement.NewClient("http://localhost:8181", polarismanagement.WithRequestEditorFn(
    func(ctx context.Context, req *http.Request) error {
        req.Header.Set("Authorization", "Bearer "+token)
        return nil
    },
))
```

Authentication uses OAuth2 client credentials against `/api/catalog/v1/oauth/tokens` (not covered by the generated client — implement separately or use `golang.org/x/oauth2/clientcredentials`).

### File naming — MUST follow this convention

| Kind | Source file | Test file |
|---|---|---|
| Resource | `resource_<type>.go` | `resource_<type>_test.go` |
| Data source | `datasource_<type>.go` | `datasource_<type>_test.go` |
| Ephemeral resource | `ephemeral_<type>.go` | `ephemeral_<type>_test.go` |

Where `<type>` is the Terraform type name without the `polaris_` prefix, with slashes replaced by underscores. Examples: `resource_catalog.go`, `datasource_realm.go`, `ephemeral_principal_credentials.go`.

### Key conventions

- Each resource file defines a struct implementing `resource.Resource` (and optionally `resource.ResourceWithImportState`).
- State models are separate structs with `tfsdk` tags.
- `provider.Configure()` passes `*ProviderData` (defined in `provider.go`) as both `resp.ResourceData` and `resp.DataSourceData`. Every resource and data source must type-assert to `*ProviderData` in their own `Configure` method.
- Acceptance tests use `resource.Test` with `providerFactories` and helpers from `provider_test.go`. Test environment variables: `TF_ACC=1`, `POLARIS_BASE_URL`, `POLARIS_CLIENT_ID`, `POLARIS_CLIENT_SECRET`, `POLARIS_REALM`.
- Tests that need a catalog call `testCreateCatalog(t)` — it creates a random catalog via the API and registers `t.Cleanup` to delete it. No `POLARIS_TEST_CATALOG` env var is needed.
- Tests that need a namespace, table, or view call `testCreateNamespace`, `testCreateTable`, or `testCreateView` — same pattern.

### Testing — MANDATORY

**You MUST write an acceptance test file whenever you add or modify a resource, data source, or ephemeral resource.** No exceptions.

- Test file lives alongside the source file: `datasource_principal_test.go` next to `datasource_principal.go`.
- Every public behaviour must be covered: the happy path, every optional/computed attribute that has non-trivial logic, and any default-value behaviour.
- Use `checkTFAcc(t)` and `testAccPreCheck(t)` at the top of every `TestAcc*` function.
- Use `testAccProviderConfig()` for the provider block — never hardcode credentials.
- Test functions are named `TestAcc<Kind><Type>_<scenario>`, e.g. `TestAccDatasourcePrincipal_defaultsToProviderClientID`.
